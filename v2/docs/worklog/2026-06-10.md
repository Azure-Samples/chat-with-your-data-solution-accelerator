## Bugs

Two ingestion defects blocked the `batch_push` pipeline from indexing any document end-to-end. Both were fixed test-first as a single allowed back-fill (Hard Rule #12), because the active task could not index a chunk without them.

* [BUG-0001](../bugs.md#bug-0001--embeddings-routed-to-the-project-endpoint-and-missing-dimensions): embeddings routed to the Foundry project endpoint (no embeddings path → `404`) and omitted `dimensions`. Fixed.
* [BUG-0002](../bugs.md#bug-0002--parser-chunk-id-is-an-illegal-azure-ai-search-document-key): parser-minted chunk id was an illegal Azure AI Search document key (`InvalidDocumentKey`). Fixed with a SHA-256 key-safe id helper.

Both are also tracked in [development_plan.md](../development_plan.md) §0.1 under `INGEST-EMBED-DOCKEY`.

## Decisions

* The defect registry [bugs.md](../bugs.md) is the single source of truth for bugs. The development plan §0.1 and §0.2 queues stay scoped to phase debt and tasks, keeping a one-line pointer row when a defect needs phase-audit visibility.
* Bug ids are sequential and zero-padded (`BUG-0001`).
* A new worklog file is created on the first meaningful action of each working day, named by the real current date.
* Historical defects already recorded in §0.1 are not back-filled into the registry now. They migrate opportunistically when next touched.
