## BUG-0013 — reasoning feed empty (design locked + Unit 1)

### Design (locked with operator)

The operator chose **model-agnostic activity narration _and_ explicit reasoning-model detection/branching** — both, not either. Governing constraints (operator): no hard-coded model names (the deployment must stay swappable), content-reactive, and account for **Foundry IQ owning retrieval server-side** (narrate from observable KB activity).

Root cause (confirmed against the post-`BUG-0021` code): the `text_reasoning → reasoning` SSE pipeline is already fully wired — `_update_to_events` maps `text_reasoning` content blocks onto `OrchestratorChannel.REASONING`, and the FE renders them in the collapsible "Thinking" panel (panel shows when `m.streaming || m.reasoning.length > 0`). The gap is upstream: `agent.run(...)` requests **no** reasoning output, so the chat model emits no `text_reasoning` summary blocks → the panel stays empty. Foundry IQ runs KB retrieval (query planning, ranking) **server-side**, so the only client-observable signals of that work are the KB `function_call` block and the returned citation annotations.

### Plan (one unit per turn, test-first, in dependency order)

* **U1 — KB-search activity narration (model-agnostic).** Emit a `reasoning`-channel narration when the agent invokes KB retrieval (the `knowledge_base_retrieve` `function_call`), immediately before the raw `tool` event for that call. Guarantees a non-empty "Thinking" panel for *any* model, independent of native reasoning summaries. **This unit.**
* **U2 — sources-found narration.** After the stream, emit a `reasoning`-channel narration summarizing the resolved citation count, in the `run()` citation section.
* **U3 — explicit reasoning-capability declaration.** Add a declared, **per-agent** reasoning-effort field to the agent-definition layer (`AgentDefinition`) — a `ReasoningEffort` `StrEnum` (`NONE`/`LOW`/`MEDIUM`/`HIGH`), defaulting `NONE`. Explicit (scenario/operator declares it), model-agnostic (no name matching), forward-compatible (swap the deployment + set the field). Per-agent (not a global `OpenAISettings` knob) so the RAI classifier is never forced to reason. CWYD declares a non-`NONE` effort; RAI stays `NONE`.
* **U4 — orchestrator branching.** When the active agent declares a non-`NONE` reasoning effort, thread the reasoning request into `agent.run(...)` via the SDK's supported reasoning option so a reasoning-capable model emits `text_reasoning` summaries; otherwise pass nothing (safe for non-reasoning models).
