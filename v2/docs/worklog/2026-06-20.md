## Summary

Took [BUG-0054](../bugs.md) from "stray poison noise" to a scoped feature build. The deployed Event Grid system topic (`evgt-<SUFFIX>`) forwards `Microsoft.Storage.BlobCreated` events into the `doc-processing` queue, but the only consumer (`batch_push`) validates CWYD ingestion envelopes, not `BlobCreated` events — so every event poisons. Research confirmed there is **no reference pattern to copy**: MACAE has no Event Grid ingestion (it batch-indexes the `data/` datasets at provision time via a script), and CWYD v1 is HTTP-driven (`BatchStartProcessing` → queue → `batch_push_results`), which v2 already ports as `batch_start` → `doc-processing` → `batch_push`. The v2 Event Grid subscription is therefore an un-ported, handler-less addition. The operator confirmed they **bulk-upload by dropping files directly into the `documents` container**, so the Event Grid path is genuinely needed — it just never had its translator built.

Decision (operator-approved): **B2 + 2a** — build the missing handler as an Event Grid *trigger* function (sidesteps the BUG-0056 queue-encoding conflict), and make Event Grid the **single** blob-ingestion trigger (admin upload writes the blob only; `batch_start` stays as reprocess-all). Documented in [ADR 0028](../adr/0028-event-grid-single-trigger-blob-ingestion.md).

## Decisions

* **B2 over B1.** An Event Grid *trigger* function deserializes the event outside the queue, so `doc-processing` stays single-schema + single-encoding and `batch_push` is untouched. B1 (dedicated `blob-events` queue) would force standardizing base64 everywhere (touching the working `batch_start` path) because the host `messageEncoding` is shared by all queue triggers.
* **2a — Event Grid is the single ingestion trigger.** Cleanest, no double-processing, no dedup hacks. Admin upload reduces to write-the-blob-only; `batch_start` stays as the explicit reprocess-all. Rollout lands the handler + bicep first (brief harmless double) and removes the backend enqueue last.
* **Delete nothing** — the subscription is repointed into a working configuration, per the operator directive.
