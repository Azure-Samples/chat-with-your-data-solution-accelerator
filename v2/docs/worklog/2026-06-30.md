## Summary

Resumed the live close-out of [BUG-0054](../bugs.md) (single Event Grid ingestion-trigger cutover). Yesterday's `azd deploy backend` had succeeded (image built+pushed, Container App revision updated, endpoint live), proving the deploy-sequencing fix. Overnight the entire resource group was emptied. Root cause is an **external account blocker**: the Azure subscription was **disabled and marked read-only** (`ReadOnlyDisabledSubscription`), so all resources were removed and no write action (provision/deploy) can run until it is re-enabled. The CWYD code, Bicep, and durable `event_grid` intent are unaffected.

## Decisions

- **BLK-01:** Stopped issuing Azure write commands. A disabled/read-only subscription is an external billing/account blocker (commonly: spending limit reached, credits expired, or admin-disabled); it cannot be resolved from code or CLI. The user must re-enable the subscription (Azure portal → Subscriptions → Reactivate).
