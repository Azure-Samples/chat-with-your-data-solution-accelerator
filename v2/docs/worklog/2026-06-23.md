## Decisions

* Backend startup failures during the restart were an **operational**
  issue, not a defect: the personal `cwyd-pgdb` Postgres flexible server
  was `Stopped`, so the shared gateway answered TCP on 5432 but dropped
  the SSL handshake (`asyncpg ... unexpected connection_lost()` inside
  `_create_ssl_connection`). Started the server (now `Ready`) and the
  backend came up clean. Not recorded as a bug ‚Äî no wrong runtime
  behavior, just a paused sandbox. Diagnostic signature captured in repo
  memory for next time.
