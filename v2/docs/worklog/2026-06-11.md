## Summary

Tracked six admin-configuration and search defects in the canonical registry after researching the v2 admin surface against the v1 parity baseline. This was a tracking-only day: no production code changed.

## Planned

* Triage the reported admin-configuration and `/api/admin/documents` issues against the actual v2 code and the v1 baseline.
* Record each confirmed defect in [bugs.md](../bugs.md) as `BUG-0003` through `BUG-0008`.
* Defer all fixes to later one-unit, test-first turns.

## Done

* Researched the v2 admin Configuration page, the orchestrator registry, the content-safety defaults, the `/api/admin/documents` route, and the v1 prompt configuration for a parity baseline.
* Added six defects to [bugs.md](../bugs.md): `BUG-0003` (documents endpoint 503), `BUG-0004` (orchestrator free-text field), `BUG-0005` (labels show config keys), `BUG-0006` (content safety default), `BUG-0007` (default prompt text), and `BUG-0008` (separate prompt editor page).

## Bugs

All six are open and tracked for fixing in later turns. No fixes landed today.

* [BUG-0003](../bugs.md#bug-0003--admin-documents-endpoint-returns-503-when-no-search-endpoint-is-configured): `GET /api/admin/documents` returns `503` when the deployment has no search endpoint, because the lifespan leaves the search provider unset.
* [BUG-0004](../bugs.md#bug-0004--orchestrator-field-is-free-text-instead-of-a-dropdown): the admin Orchestrator field is free text; it should be a dropdown of the known orchestrator keys.
* [BUG-0005](../bugs.md#bug-0005--admin-configuration-labels-show-internal-config-key-names): the admin Configuration labels show internal config-key names such as `(orchestrator_name)`.
* [BUG-0006](../bugs.md#bug-0006--content-safety-defaults-to-disabled): Content Safety defaults to disabled; it should default to enabled.
* [BUG-0007](../bugs.md#bug-0007--default-agent-instructions-do-not-carry-the-v1-default-prompt-text): the default agent instructions do not carry the vetted v1 default prompt text.
* [BUG-0008](../bugs.md#bug-0008--separate-prompt-editor-page-should-fold-into-configuration): the separate Prompt editor page should be removed and folded into the Configuration page, matching v1.

## Decisions

* Post-answering is not a defect. Research confirmed v1 ships a full post-answering feature (a default prompt, a fact-checking validator, and an operator filter message), so the v2 post-answering trio matches v1 and stays as is.
* Bringing v1 prompts into v2 is scoped to the default prompt text only. The v1 per-business-unit assistant-type system (Default, Contract assistant, Employee assistant) is out of scope for v2.
* All six items were recorded in [bugs.md](../bugs.md) rather than split across the development plan debt queues, by request. No `BUG-####` pointer rows were added to the development plan this turn; they can be added later if a defect needs phase-audit visibility.

## Next

* Fix the defects one unit per turn, test-first.
* Start with `BUG-0003`: decide whether the affected environment should have a search endpoint or whether the documents endpoint should degrade gracefully instead of returning `503`.
